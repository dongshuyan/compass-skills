# HTML 可视化契约

## 目标

`task-forest.html` 是用户理解任务森林的主要界面。它必须让用户在不打开 JSON 的情况下看清：

- 当前任务主层级；
- 完整 DAG 关系；
- 哪些任务未完成、阻塞或待复核；
- 每次 session 如何改变任务图；
- 每个节点的目的、要求、验收、风险、偏差和关系边。

导出 HTML 必须是离线、无外部依赖、可分享的静态文件。交互状态只存在于当前浏览器页面中，不能写回 canonical graph。

## 必备视图

### 树视图

- 默认使用 `child_of` 生成主层级树。
- 树应竖向分层：根节点在上，子节点在下。
- 节点卡片只展示短标题和进度条；详细说明放在右侧详情面板。
- 节点必须可点击，点击后右侧显示完整节点详情。
- 支持全部展开、全部折叠、搜索和状态筛选。

### DAG 视图

- 按完整 DAG 展示所有可见节点和关系边。
- 关系边必须按类型使用稳定颜色，并带可点击中文标签。
- `child_of` 在用户界面中显示为 `子任务`，不要显示“父子分解”。
- 边标签应跟随对应边的位置；拖动节点时，边路径和边标签必须同步移动。
- DAG 节点可单独拖拽，用于整理视图；拖拽位置只保存在页面运行态，不能写入任务图。
- 必须提供 `重置布局`，清空运行态拖拽位置并恢复自动布局。

### 历史变化

- 使用 snapshots 展示任务图如何随 session 增长或变化。
- 支持滑块手动查看历史。
- 支持播放、暂停、上一步、下一步、循环和播放速度选择。
- 每个快照应展示新增节点、新增边、状态变化和进度变化摘要。

### 状态队列和 todo

- 左侧必须显示待复核、阻塞、可执行、持续目标和未完成事项。
- 顶部统计中的 `待复核`、`阻塞`、`未完成` 应可点击筛选，再次点击应取消筛选。
- `review_needed` 不能只显示数字；用户必须能点到具体节点并看到复核说明。

### 节点详情

- 右侧详情必须展示：
  - 状态、类型、进度、优先级；
  - 全局目的；
  - 为什么该任务符合目的；
  - 不足或风险；
  - 期望结果、成功指标、具体要求、验收标准、非目标、关键假设；
  - 验证计划、执行提示、证据、关系边、偏差记录。
- `review_needed` 节点必须展示“待复核要看什么”，包括复核对象、通过复核如何生成 proposal、不通过复核如何记录缺口。
- 复核说明应提供可复制的通过/不通过指令，但不能直接在 HTML 中修改正式任务图。

### 边详情

- 所有边都必须可点击。
- 点击边后右侧显示边详情，包括方向、起点任务、目标任务、类型、阻塞属性、置信度、原因和创建信息。
- 边详情应有“返回上一项”和“返回当前节点”入口，避免用户迷失上下文。

## 布局和交互

- 左右面板默认展开，允许收起。
- 左右面板收起后应保留窄侧栏；页面向下滚动时，折叠按钮和竖排标题仍必须保持在视口内。
- 画布支持拖拽空白区域平移。
- 按住 Ctrl、Command 或 Alt 并滚轮可缩放。
- 提供缩小、放大、重置缩放、适配画布按钮。
- 首次打开或切换视图后，应尽量适配画布，让用户能看到整体结构。
- 搜索、筛选和工具栏不能遮挡任务图。

## 中文和术语

界面面向中文用户时，核心标签应使用中文：

```text
树视图
DAG 视图
全部
全部展开
全部折叠
清除筛选
重置布局
历史与队列
节点详情
边详情
子任务
依赖
贡献
澄清
待复核
阻塞
可执行
持续目标
```

避免在用户界面直接暴露内部术语，除非它是节点 ID、边 ID 或字段名。特别是：

- `child_of` 对用户显示为 `子任务`；
- `review_needed` 对用户显示为 `待复核`；
- `global_task` 对用户显示为 `全局任务`。

## 数据边界

- HTML 读取 `exports/task-forest.graph.json` 等价数据的内嵌快照，但不是权威存储。
- 权威状态仍是 `graph/nodes.json` 和 `graph/edges.json`。
- HTML 中的搜索、筛选、缩放、平移、DAG 节点拖拽、折叠面板、历史播放都是视图状态。
- HTML 不能直接 apply proposal、改状态、写节点或写边；它只能生成可复制指令，引导用户回到 `$task-forest` 走 proposal 流程。

## 从零验收

从空 workspace 验证时，至少应覆盖：

- 初始化空 task-forest；
- 创建一个 evergreen / in_progress 顶层目标；
- 创建 `review_needed`、`ready`、`done`、`risk`、`follow_up` 等不同状态和类型节点；
- 创建 `child_of`、`depends_on`、`contributes_to`、`clarifies` 等多种边；
- 生成至少两个快照，使历史播放有内容；
- 运行 `validate`；
- 运行 `export`；
- 检查 HTML 中存在树视图、DAG 视图、可点击边、DAG 节点拖拽、标签跟随、重置布局、待复核说明、历史播放、sticky 折叠侧栏和中文术语。

推荐命令：

```bash
python3 scripts/validate_task_forest_export.py --skill-dir /path/to/task-forest
```
