# Anti Profile Bubble

长期画像和当前交互估计会提高效率，也可能把用户固定在旧模式里。每次使用画像或强假设时，做一次轻量反茧房检查。

## Checks

- 当前用户是否明确覆盖了历史偏好？
- 当前领域是否不同于用户熟悉领域？
- 当前任务风险是否比常规任务更高？
- 是否存在一个看似不符合画像但更符合当前证据的解释？
- 是否有一个遗漏项会让结果失败：受众、验收、非目标、证据边界、安全边界？

## One Out-Of-Profile Question

重大任务中至少保留一个跳出固定模式的问题，除非 contract 已完全覆盖：

- "我按你通常的低打扰偏好可以直接推进；但这次涉及 [风险]，是否需要先锁定验收/回滚/确认点？"
- "我当前按 [画像/上下文] 理解为 X。有没有一种情况是你这次反而想要 Y？"
- "这个任务是否有一个你没有明说、但会让结果成败反转的受众或限制？"

不要为了反茧房而问空泛问题。问题仍必须 decision-changing。
