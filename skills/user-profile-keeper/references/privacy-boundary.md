# Privacy Boundary

本 skill 的隐私目标是：本地保存、最小必要、可检查、可删除、可纠错、默认不扩散。

## Hard Rules

- 不联网、不上传、不调用外部 API。
- 不读取浏览器 cookie、token、keychain、SSH key、密码管理器或私有凭据。
- 不保存 secret 原文。发现 secret 时只记录 redaction 事件，例如“用户提到某凭据存在，但内容未保存”。
- 完整画像不提供给普通 skill。其他 skill 只能读取低敏 `clarification_summary`。
- 不把外部网页、第三方文件或工具输出当作用户画像偏好来源，除非用户明确确认。
- 不把用户的年龄段、教育、专业、职业、长期目标或所在地推断暴露给其他 skill；这些属于本 skill 内部维护的背景上下文。

## Sensitive Information

以下内容默认 pending 或 redacted：

- 健康、心理、创伤、亲密关系、家庭、财务、政治、宗教、身份、法律风险。
- 羞耻、低道德、违法边界或可能让用户被不公平判断的信息。
- 可定位用户真实身份、组织、客户、未公开项目或敏感会议的细节。
- 真实所在地、家乡、国籍、民族、政治或宗教等身份相关推断。

可以保存“协作上必要的低敏抽象”，例如：

- “某些敏感主题上用户可能倾向模糊表达，追问时要非评判、允许不回答。”

不要默认保存具体敏感细节。

## Background Information

用户主动提供的背景信息可以帮助长期协作，但默认较高隐私：

- 年龄段、学历、专业、职业/角色、经验阶段、长期目标默认 `private`。
- 只有用户明确要求这些信息用于一般澄清摘要，才可考虑降为 `low`；降级也应通过 proposal。
- 真实姓名、精确生日、身份证明、家庭地址、具体单位、客户、未公开项目、会议细节等不应作为普通背景画像保存。
- 背景信息不得用于限制用户请求、评价用户能力或替代当前任务证据。

## Location And Identity Inference

不得仅凭以下信号推断或保存“用户是哪里人”：

- 使用中文、英文或某种表达习惯。
- 当前时区、系统 locale、路径名、用户名、文件名。
- 工作目录、项目名、浏览器状态或联网位置。

如果用户明确要求 agent 猜测某类背景信息，只能生成带置信度、反证和确认问题的 pending proposal；不得自动写入 active。缺少实用价值或容易造成定型时，应直接跳过推断。

## Local Security

- 默认本地目录 `<home>/.compass-skills/user-profiles/v1` 使用 `0700`；可用 `COMPASS_USER_PROFILE_HOME` 指向其他本地目录。
- SQLite 数据库使用 `0600`。
- 画像默认明文存储，不做内置加密；本 skill 的安全边界是不联网、不上传、不读取 credential、最小扩散和可删除，而不是抵御同机恶意进程、系统管理员、磁盘取证或未加密备份。
- 导出文件默认放在用户目录下的 `exports/`，并支持 redacted 导出。
- `.git` 工作区内不得自动复制完整画像。
- 可选加密可后续接入 macOS Keychain、age 或 SQLCipher，但不得强制安装依赖。

## User Rights

用户可以要求：

- 查看完整画像。
- 只查看低敏摘要。
- 修改某条断言。
- 撤回某条断言。
- 删除某类信息。
- 导出脱敏副本。
- 清空整个用户画像。
